## Internationalisation & RTL Guide

- **Resources:** Strings live under `src/i18n/resources/<locale>/*.json`. Keep namespace parity across locales.
- **ICU Formatting:** Use ICU message syntax (`{count, plural, ...}`) to handle numbers and plurals.
- **Providers:** `AppProviders` wraps every route with `I18nProvider`. Use `useTranslation(["namespace"])` inside components.
- **Locale Switching:** `LocaleSwitcher` component displays a button with the opposite language indicator (shows "Ø¹" when English is active, "en" when Arabic is active). Clicking toggles between locales and updates the pathname, persisting the choice in cookie `tm.locale`.
- **RTL Support:** 
  - `I18nProvider` sets `document.documentElement.dir` based on locale. Use logical Tailwind classes (padding-start/end) where possible or rely on CSS variables.
  - Select dropdowns automatically flip arrow position for RTL (CSS in `globals.css` handles this).
  - DatePicker component fully supports RTL with proper calendar positioning and navigation.
- **Formatting Helpers:** Prefer `formatDate` utilities or `Intl.DateTimeFormat` with the current locale from `useCurrentLocale`.
- **Date Components:** The `DatePicker` component (`src/ui/components/DatePicker.tsx`) uses `react-day-picker` with full RTL and i18n support, automatically formatting dates according to the current locale.
